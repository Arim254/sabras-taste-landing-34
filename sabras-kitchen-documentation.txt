1. COLOR SYSTEM & DESIGN TOKENS
==============================

PRIMARY COLOR PALETTE (HSL Format):
----------------------------------
- Background: hsl(0 0% 100%) - Pure white for main background
- Foreground: hsl(222.2 84% 4.9%) - Very dark blue for text
- Primary: hsl(var(--primary)) - Main brand color for buttons, headings
- Primary Foreground: hsl(var(--primary-foreground)) - Text on primary background
- Secondary: hsl(var(--secondary)) - Supporting brand color
- Muted: hsl(var(--muted)) - Subtle background colors
- Accent: hsl(var(--accent)) - Highlight colors
- Border: hsl(var(--border)) - Border colors for cards and dividers
- Card: hsl(var(--card)) - Card background colors
- Destructive: hsl(var(--destructive)) - Error/warning colors

DARK MODE PALETTE:
-----------------
- Background: hsl(222.2 84% 4.9%) - Dark background
- Foreground: hsl(210 40% 98%) - Light text
- Card: hsl(222.2 84% 4.9%) - Dark card backgrounds
- Primary: Maintains brand consistency in dark mode
- Muted: hsl(217.2 32.6% 17.5%) - Dark muted backgrounds

CUSTOM DESIGN TOKENS:
--------------------
- --gradient-warm: Linear gradient for warm background effects
- --shadow-warm: Warm shadow for cards and buttons
- --shadow-hover: Enhanced shadow on hover states
- --radius: 0.5rem - Consistent border radius throughout

COLOR USAGE BREAKDOWN:
---------------------
- Hero Section: Primary colors with overlay effects
- Navigation: Foreground text with primary accent on active states
- Cards: Card background with border accent
- Buttons: Primary background with primary-foreground text
- Text Hierarchy: Foreground for body text, primary for headings, muted for secondary text
- Backgrounds: Gradient-warm for section variations

2. LAYOUT STRUCTURE & POSITIONING
=================================

OVERALL LAYOUT ARCHITECTURE:
---------------------------
The website follows a single-page application structure with smooth scrolling navigation:

HEADER (Fixed Position):
-----------------------
- Position: Fixed top, z-index high
- Width: Full width (100vw)
- Height: Auto (approximately 80px)
- Background: Semi-transparent with backdrop blur
- Content: Left-aligned logo, center navigation (desktop), right CTA button
- Mobile: Hamburger menu replaces center navigation

HERO SECTION:
------------
- Height: Full viewport (min-h-screen)
- Background: Full-cover background image with overlay
- Content Positioning: Center-aligned both horizontally and vertically
- Z-index layering: Background image (z-0), overlay (z-5), content (z-10)
- Scroll indicator: Absolute positioned at bottom center

ABOUT SECTION:
-------------
- Padding: 5rem top/bottom (py-20)
- Background: Gradient warm background
- Content: Max-width container (max-w-4xl) centered
- Card: Semi-transparent background with backdrop blur

SERVICES SECTION:
----------------
- Layout: Grid system (grid-cols-1 md:grid-cols-2 lg:grid-cols-3)
- Gap: 2rem between cards (gap-8)
- Card positioning: Each service card is self-contained with icon, content, and CTA

GALLERY SECTION:
---------------
- Grid: 2 columns mobile, 3 columns desktop (grid-cols-2 md:grid-cols-3)
- Aspect ratio: Maintained for all images
- Hover effects: Transform and overlay animations

TESTIMONIALS SECTION:
--------------------
- Grid: Single column mobile, 2 columns tablet, 3 columns desktop
- Star ratings: Flexbox layout with consistent spacing
- Cards: Equal height with flexbox stretch

CONTACT SECTION:
---------------
- Layout: Two-column grid (md:grid-cols-2)
- Left: Contact form in card container
- Right: Contact information and quick actions
- Gap: 3rem between columns (gap-12)

FOOTER:
------
- Layout: Three-column grid on desktop, stacked on mobile
- Positioning: Full width with centered content container
- Social links: Horizontal flexbox layout

BUTTON POSITIONING PATTERNS:
---------------------------
- Hero CTA: Center-aligned with generous padding
- Service CTAs: Bottom-aligned within each card
- Form submit: Right-aligned within form container
- Navigation CTAs: Right-aligned in header
- Contact actions: Vertically stacked in contact info cards

CARD POSITIONING:
----------------
- Service Cards: Grid layout with equal spacing
- About Card: Single centered card with max-width constraint
- Contact Cards: Mixed layout - form card spans full height, info cards stacked
- Testimonial Cards: Grid with equal heights using flexbox
- Gallery Cards: Masonry-style grid with hover effects

6. ANIMATION & TRANSITIONS
==========================

ANIMATION LIBRARY:
-----------------
- CSS transitions for smooth state changes
- Transform animations for interactive feedback
- Keyframe animations for loading states
- Scroll-triggered animations (bounce indicator)

TRANSITION TIMINGS:
------------------
- Fast interactions: 0.15s (button hovers)
- Standard transitions: 0.3s (card hovers, form states)
- Slow animations: 0.5s (page transitions, complex movements)
- Easing: cubic-bezier(0.4, 0, 0.2, 1) for natural feel

SPECIFIC ANIMATIONS:
-------------------

1. Hero Section:
   - Bounce animation for scroll indicator
   - Fade-in effects for content
   - Background parallax subtle movement

2. Navigation:
   - Mobile menu slide-in from right
   - Link underline expansion animations
   - Logo hover effects

3. Cards:
   - Lift effect on hover (translateY and box-shadow)
   - Content fade-in on page load
   - Staggered animations for card grids

4. Buttons:
   - Scale transform on hover (1.05x)
   - Color transitions for background and text
   - Active state feedback (0.95x scale)

5. Form Elements:
   - Focus ring animations
   - Border color transitions
   - Validation feedback animations

6. Gallery:
   - Image zoom on hover
   - Overlay slide-up animations
   - Aspect ratio maintenance

PERFORMANCE CONSIDERATIONS:
--------------------------
- Hardware-accelerated animations (transform, opacity)
- Minimal repaints and reflows
- Optimized animation properties
- Reduced motion support for accessibility
